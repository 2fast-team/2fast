| TutorialContentDialogHowTheAppWorks.Text                           | A funcionalidade para a [Autenticação](https://en.wikipedia.org/wiki/Authentication) do segundo fator é feita através da palavra-passe de uso único baseada no tempo. Para este efeito, é acordada uma chave de configuração secreta entre o servidor de início de sessão do serviço e o utilizador, que é armazenada localmente na aplicação. Isto garante que, se a chave de configuração for acordada, a aplicação pode sempre ser utilizada offline.



O armazenamento das chaves de configuração para os vários serviços nesta aplicação, ao contrário de outros autenticadores, é armazenado num ficheiro, que é protegido por palavra-passe na criação para encriptar o ficheiro. Para armazenar o ficheiro, recomenda-se, neste caso, um armazenamento na nuvem, como Nextcloud, ownCloud, Onedrive, etc., a fim de obter um controlo de versão e segurança contra perdas.
